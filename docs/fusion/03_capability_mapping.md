# 03 能力映射

## 风格映射矩阵（SilverM -> xy_quant）

### 目标
把 SilverM 的工程风格映射为 xy_quant 融合计划中的可执行能力要求，明确哪些能力必须继承，哪些能力需要按 xy_quant 现状做等价实现。

### 规则
- SilverM 的“统一数据接口”映射到 xy_quant 的数据接入层、指标层、回测层，必须保持接口一致性或兼容适配层。
- SilverM 的“多数据源健康检查 + 断路器”映射到 xy_quant 的数据源管理，必须具备健康探测、失败切换、恢复回切。
- SilverM 的“回测引擎 + 报告生成”映射到 xy_quant 的回测与结果产出链路，必须支持可复盘结果、运行ID、报告导出。
- SilverM 的“因子分析 + 多因子策略”映射到 xy_quant 的策略研究链路，必须支持因子统计、分层、打分与回测闭环。
- SilverM 的“dashboard/API”映射到 xy_quant 的可观测层，必须保留查询、展示、审计的最小闭环。

### 检查清单
- [ ] 数据接口是否存在统一入口或兼容适配层。
- [ ] 数据源是否具备健康检查、失败切换和恢复回切逻辑。
- [ ] 回测结果是否可通过 run_id、报告文件或数据库记录复现。
- [ ] 因子分析是否支持 IC、分位数收益或相关性检查。
- [ ] 是否存在可查询、可展示、可审计的结果输出层。

### 验收标准
- 每项 SilverM 能力都能在 xy_quant 中找到对应模块或明确的迁移实现。
- 每项映射都能用文件路径、函数名或表名进行验证。
- 不允许出现仅写“对标”“参考”“借鉴”但没有落地路径的条目。

| SilverM 能力 | 证据文件 | xy_quant 目标能力 | 验证方式 |
|---|---|---|---|
| 统一数据接口 | shared/data/README.md | 数据接入与适配层统一 | 检查适配器入口与返回格式 |
| 回测引擎 | backtest/engine.py | 回测执行与指标输出 | 检查 run_id、分析器、数据源 |
| 因子分析 | tools/analysis/factor_analysis.py | 因子统计与筛选 | 检查 IC/分位数/相关性函数 |
| 报告生成 | tools/visualization/generate_report.py | 回测可视化与导出 | 检查报告文件输出路径 |
| 健康检查 | scripts/data_check/health_check.py | 数据源监控与切换 | 检查响应时间与可用性 |
| 风险控制 | data/fetchers/multi_source_fetcher.py | 断路器与故障转移 | 检查失败率阈值与状态机 |
