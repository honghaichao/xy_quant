# xy_quant 融合计划（SilverM 风格对齐）

## SilverM风格提炼与对齐总则

### 目标
将 SilverM 的工程实践提炼为可执行、可核验的约束，统一落地到 xy_quant 融合计划，确保后续文档、实现和验收标准一致。

### 规则
- 以“能跑、能查、能回滚”为第一原则，所有约束必须指向具体目录、脚本、数据表或校验动作。
- 优先继承 SilverM 已验证的工程风格：统一数据接口、显式回测边界、明确风控阈值、可观测日志与报告输出。
- 所有计划描述必须能被文件路径、函数名、表名、命令或检查项直接验证。
- 不写抽象口号，不引入无法验收的“提升”“优化”“增强”等空泛表述。

### 检查清单
- [ ] 每条风格约束都能在 SilverM 证据文件中找到对应实现或说明。
- [ ] 每条约束都能映射到 xy_quant 的具体文档条目或后续开发任务。
- [ ] 文档中的规则均包含检查动作，而不是仅描述目标。
- [ ] 文档中没有引入 SilverM 未体现的额外工程假设。

### 验收标准
- 文档中的风格提炼可被逐条追溯到 SilverM 的实际目录与文件。
- xy_quant 融合计划中的约束可直接用于评审、开发和验收。
- 任意一条新增规则都能通过“路径 + 规则 + 检查项”三元组核验。
